# Instagram ローカル投稿プレビューアプリ — 設計書

作成日: 2026-06-14

## 1. 目的

インスタグラムに投稿する前に、「フィード投稿」「ストーリー」「プロフィールのグリッド」が
**投稿後にどう見えるか**をローカルで確認するためのアプリ。
実際の投稿はせず、手元（ブラウザ）で見た目と挙動を忠実に再現する。

> ユースケース: 写真やキャプションを入れて「シェア」すると、本物のインスタを開いているように
> フィード・プロフィール・ストーリーに反映され、投稿後の見え方を事前にチェックできる。

## 2. スコープ

### やること
- インスタそっくりのダークUI（アプリ全体っぽく：上部バー＋下部タブ）
- 4つの画面: ホーム/フィード、ストーリー全画面ビューア、プロフィール（グリッド）、投稿作成
- 投稿メディア: **写真1枚 / 複数枚カルーセル / 動画**（インスタと同等）
- プロフィール（ユーザー名・表示名・アイコン・bio）を自分で編集できる
- 投稿はIndexedDBに保存し、リロードしても消えない
- すべてローカル完結（外部通信・ログイン一切なし）

### やらないこと（割り切り / YAGNI）
- 本物のアカウント連携・ネットワーク通信・サーバー
- 他ユーザーからの本物のいいね/コメント/フォロー（数字は自分で任意に設定できる程度）
- 検索結果ページ・DM（メッセージ）・リールのフィード/アルゴリズム
  - ※DMとリールは今回のプレビュー対象に含まれないため作らない
- フィルター/画像編集ツール、複数アカウント切替

## 3. 技術スタック

- **React + Vite**（SPA、ビルド/devサーバーは Vite）
- **プレーンCSS**（CSS Modules も可）— インスタの見た目をピクセル単位で寄せるため、
  Material 等のUIライブラリは使わない
- **IndexedDB** によるローカル永続化（メディアBlob＋メタデータ）。
  外部依存を増やさないため薄い自前ラッパーを実装する（`idb` パッケージ採用も可、実装計画で確定）
- 状態管理: React Context + カスタムフック（ルーターや Redux/Zustand などの追加依存は使わない）
- アイコン: インラインSVGでインスタのライン系アイコンに寄せる
  （Instagram の商標ロゴ画像そのものは同梱せず、見た目を再現する範囲にとどめる）

## 4. 画面構成

下部タブで切り替える「アプリ全体」構成。画面遷移は store の `currentTab` ＋
オーバーレイ（ストーリービューア・投稿作成・投稿詳細）で管理し、react-router は使わない。

### 4.1 ホーム / フィード（フィード投稿の見え方）
- 上部バー: 左にロゴ風ワードマーク、右にDM/＋ などのアイコン（飾り）
- ストーリーのトレイ（横スクロール）: 「自分のストーリー」＋投稿済みストーリーのサムネ。タップで 4.2 を開く
- 投稿リスト（feed種別の投稿を新しい順）。各投稿カード `PostCard`:
  - ヘッダー: アイコン・ユーザー名・「…」メニュー
  - メディア: 写真 / カルーセル（ドット付き） / 動画
  - アクション行: いいね・コメント・シェア・（右に）保存
  - いいね数 → キャプション（ユーザー名＋本文）→ コメントを見るリンク（飾り）→ 投稿時刻

### 4.2 ストーリー全画面ビューア（ストーリーの見え方）
- 全画面（縦長）。上部に進行バー（複数ストーリーぶん）
- ヘッダー: アイコン・ユーザー名・経過時間・「×」閉じる
- メディア表示。自動送り（画像=5秒 / 動画=尺ぶん）、左右タップで前後送り、長押しで一時停止
- 下部に返信バー（飾り）・いいねアイコン

### 4.3 プロフィール（グリッドの見え方）
- 上部バー: ユーザー名
- ヘッダー: アイコン・投稿数/フォロワー/フォロー数・表示名・bio・「プロフィールを編集」ボタン
- 3列グリッド: feed投稿のサムネ（カルーセルは複数枚マーク、動画は再生マーク）。タップで投稿詳細
- 「プロフィールを編集」→ プロフィール編集画面（ユーザー名・表示名・アイコン画像・bio）

### 4.4 投稿作成（＋）
- 種別選択: 「フィード投稿」or「ストーリー」
- メディア選択: 1枚 / 複数（カルーセル、並び替え可）/ 動画
- フィードのみ: キャプション入力（任意でいいね数・場所などの軽い項目）
- 「シェア」→ store に保存 → フィード/プロフィール/ストーリーに即反映

## 5. データモデル（IndexedDB）

```
profile (単一レコード)
  username: string
  displayName: string
  avatarBlob: Blob | null
  bio: string

posts (複数レコード)
  id: string (uuid)
  type: 'feed' | 'story'
  media: Array<{ kind: 'image' | 'video', blob: Blob }>   // 順序＝表示順
  caption: string        // feed のみ使用
  likes: number          // 任意設定
  location?: string      // 任意
  createdAt: number      // epoch ms（並び順に使用）
```

- feed投稿: フィード一覧＋プロフィールグリッドに表示
- story投稿: ストーリートレイ＋ストーリービューアに表示（自動失効はしない＝プレビュー用途のため残す）

## 6. メディアの扱い

- 入力: `<input type=file>` / ドラッグ&ドロップ → `File`(Blob) を取得
- 保存: Blob を IndexedDB にそのまま格納（容量制約の大きい localStorage は使わない）
- 表示: レンダリング時に `URL.createObjectURL(blob)` でURL生成、アンマウント時に `revokeObjectURL`
- 動画: `<video>` で再生。フィードはミュート自動再生（インスタ準拠）、ストーリーは尺で自動送り

## 7. 主要なインタラクション

- カルーセル: 横スワイプ/ドラッグ＋下部ドットインジケータ
- ストーリー: 進行バーの自動進行、左右タップ送り、長押し一時停止
- いいね: タップでハート切替＋軽いアニメ（ダブルタップでいいねも）
- 下部タブナビゲーション（ホーム/検索(飾り)/＋/リール(飾り)/プロフィール）
- ダークテーマでインスタの配色・余白・フォントサイズを再現

## 9. 実行方法

- 開発: `npm install` → `npm run dev` → 表示されたローカルURLをブラウザで開く
- ビルド（任意）: `npm run build` → `npm run preview` で静的配信を確認

## 10. デザイン方針

- 参考画像のダークなインスタUIに忠実（黒背景、白文字、グレーの補助色、青のアクセント）
- フォントはシステムフォントでインスタの字面に近づける
- アイコンはSVGで近似（商標ロゴ画像そのものは同梱しない）
- スマホ幅（〜430px）を基準にレイアウト、PCブラウザでは中央に端末幅で表示

## 11. テスト方針

- データ層（`db/db.js`／store アクション）は Vitest でユニットテスト
  （投稿の追加・取得順・プロフィール編集・グリッド/フィード反映）
- UIは実際にアプリを起動して各画面を目視確認＋スクリーンショット
  （投稿作成→フィード/プロフィール/ストーリー反映の一連フロー）

## 12. リスク・割り切り

- IndexedDB の容量はブラウザ依存（多くは数百MB〜）。大量・長尺動画は実用域で運用
- 「インスタと全く同じ見た目」は商標ロゴ等を除き、レイアウト/配色/挙動の再現で達成する
- 本ツールは個人がローカルで自分の投稿の見え方を確認する用途に限定する
